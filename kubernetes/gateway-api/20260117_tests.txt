I had to install traefik without applying the CRDs included in the chart, because I already have Gateway API CRDs installed in my cluster (by another API Gateway).

--skip-crds for avoiding to download all the traefik resources (i got a gaetaway api CDRs custom)
helm install traefik traefik/traefik `
  --version $CHART_VERSION `
  --values kubernetes/gateway-api/values.yaml `
  --skip-crds `
  --namespace traefik `
  --create-namespace

To fix this i fixedthe kubernetesCRD installation in the values.yaml file of traefik :
providers:
  ...
  kubernetesCRD:
    enabled: false  
helm upgrade traefik traefik/traefik `
  --version $CHART_VERSION `
  --values kubernetes/gateway-api/values.yaml `
  --skip-crds `
  --namespace traefik

after upgrade:
- Delete the traefik pod to force re-creation: kubectl delete pod -n traefik <podname>
- port forwarding again: kubectl port-forward -n traefik svc/traefik 8080:80
- check the logs again: kubectl logs -n traefik <podname>
